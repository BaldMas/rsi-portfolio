# Dropstab Portfolio Rotation

Автоматический трекер ротации крипто-портфеля с HTML-отчётом на GitHub Pages.

**Отчёт:** https://baldmas.github.io/rsi-portfolio/

---

## Что делает система

1. Каждые 15 минут подтягивает актуальный портфель из Dropstab API
2. Рассчитывает RSI, Z-score, RS30, MA200, OI, Funding для каждой пары монет
3. Генерирует HTML-отчёт и публикует на GitHub Pages
4. Каждый час собирает новости по монетам (Google News RSS, 4 языка)
5. 3 раза в день отправляет рекомендации в Telegram (09:00 / 15:00 / 21:00 MNE)
6. Отслеживает изменения состава портфеля (куплено / продано / пополнено)
7. При появлении новой монеты - автоматически запускает её анализ

## Настройка (.env)

```
DROPSTAB_TOKEN=<токен из URL публичной страницы Dropstab>
TELEGRAM_TOKEN=<токен бота от @BotFather>
TELEGRAM_CHAT_ID=<chat_id группы или пользователя>
```

### Как получить Telegram chat_id

1. Создай бота через @BotFather, получи токен
2. Добавь бота в нужную группу
3. Отправь `/chatid` в группе - бот ответит chat_id
4. Вставь в `.env`

## Колонки отчёта

| Колонка | Что показывает |
|---------|---------------|
| **Из → В** | Пара монет для ротации. Под каждой монетой: место в CoinMarketCap + P/L%. Под получателем - P/L% после ротации |
| **RSI 1D / 1W** | RSI дневной и недельный по паре (цена А / цена Б) |
| **Z-score 90д** | Отклонение текущего соотношения от 90-дневной средней |
| **RS 30д** | Relative Strength: насколько монета А опережает/отстаёт от Б за 30 дней |
| **MA200** | Расстояние текущей цены от 200-дневной MA в % |
| **OI 7д** | Изменение открытого интереса за 7 дней (фьючерсы) |
| **Funding** | Ставка финансирования (последняя / средняя за 30д) |
| **Сигнал** | Итоговый скор ротации. >4 = ротировать, 2-4 = смотреть, <-3 = ждать |
| **Инвест** | Сколько вложено в монету-источник |
| **Сейчас (P/L)** | Текущая стоимость позиции + P/L% от вложенного |
| **Доход протокола** | Fees/Revenue за 30д по DeFiLlama. Бейдж: влияет / косвенно / не влияет на цену |
| **Статус** | Кнопка-фильтр: РОТИРОВАТЬ / смотреть / ждать |

## Правило ротации капитала

Деньги могут перетекать только в равную или лучшую группу:
- G5 → G1, G2, G3, G4, G5
- G4 → G1, G2, G3, G4
- G3 → G1, G2, G3
- G2 → G1, G2
- G1 → G1 только

Группы монет хранятся в `/home/masus/crypto_analysis/_portfolio_groups.md`

## Скоринг монет (29 критериев, 0-500 баллов)

Файлы анализа: `/home/masus/crypto_analysis/<ТИКЕР>.md`

Веса по уровням:
- S-тир (критические): ×8, ×7, ×7, ×7, ×3, ×4
- A-тир (важные): ×7, ×2, ×2, ×1, ×2, ×2, ×7, ×1, ×2, ×7, ×2
- B-тир (дополнительные): ×3, ×1, ×2, ×3, ×1, ×4
- C-тир (контекст): ×7, ×4, ×4, ×3, ×1, ×1

7 хард-фильтров (F1-F7): F1 объём >$20M, F2 MCap >$200M, F3 Circ/FDV >25%, F7 (ручной). F1/F2/F3/F7 = автоматический FAIL.

## Cron (каждые 15 минут)

```
*/15 * * * * cd /home/masus/dropstab && bash update_report.sh >> update.log 2>&1
```

- Каждые 15 мин: генерация HTML + git push
- Каждый час (в :00 и :15): сбор новостей
- 07:00 / 13:00 / 19:00 UTC (= 09:00 / 15:00 / 21:00 MNE): Telegram рекомендации

## Добавление новой монеты

1. Монета автоматически появится в отчёте, как только попадёт в Dropstab портфель
2. Запустится автоанализ (auto_analyze.py) - 7 из 29 критериев заполнятся автоматически
3. Оставшиеся 22 критерия нужно заполнить вручную в `/home/masus/crypto_analysis/<ТИКЕР>.md`
4. Обновить группу в `/home/masus/crypto_analysis/_portfolio_groups.md`

## Обновление скоринга

Рекомендуемая частота пересмотра:
- **G1-G2**: раз в квартал + по событию
- **G3**: раз в 6 недель
- **G4-G5**: раз в месяц
